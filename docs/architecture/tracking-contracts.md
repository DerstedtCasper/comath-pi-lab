# Tracking Contracts

## TODO.md

`TODO.md` is phase-scoped and checkbox-driven.

Required sections:

- current phase tasks;
- upcoming phase seed tasks;
- known deferred items;
- blockers if any.

## REVIEW.md

`REVIEW.md` is the phase handoff record.

Required sections:

- scope;
- runtime assumptions;
- external boundary notes;
- validation commands and outcomes;
- changed files;
- risks;
- next phase readiness.

## AGENTS.md

`AGENTS.md` defines repository-local agent behavior.

Required sections:

- current phase;
- required reading;
- write boundaries;
- proof/evidence rules;
- phase boundary requirements.
