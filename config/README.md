# Config Samples

This directory contains non-secret configuration examples. Do not commit live credentials, API keys, private paths, paid-provider account details, or user paper corpora.

Runtime configuration is owned by `comathd`; Pi may select declared options but must not receive secret values.

Adapter OS-isolation provider helpers are configured outside the sample with absolute service-owned executable paths. Provider-specific executable handles are `COMATH_AGENT_ADAPTER_OSISO_OCI_HELPER`, `COMATH_AGENT_ADAPTER_OSISO_NIX_HELPER`, `COMATH_AGENT_ADAPTER_OSISO_FIREJAIL_HELPER`, `COMATH_AGENT_ADAPTER_OSISO_WINDOWS_APPCONTAINER_HELPER`, and `COMATH_AGENT_ADAPTER_OSISO_MACOS_SANDBOX_EXEC_HELPER`. A generic fallback executable handle, `COMATH_AGENT_ADAPTER_OSISO_PROVIDER_HELPER`, may be used only as host configuration when no provider-specific helper variable is set.

Optional fixed helper execution asset arguments may be configured as bounded JSON string arrays through `COMATH_AGENT_ADAPTER_OSISO_OCI_HELPER_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_NIX_HELPER_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_FIREJAIL_HELPER_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_WINDOWS_APPCONTAINER_HELPER_ARGS_JSON`, or `COMATH_AGENT_ADAPTER_OSISO_MACOS_SANDBOX_EXEC_HELPER_ARGS_JSON`, with `COMATH_AGENT_ADAPTER_OSISO_PROVIDER_HELPER_ARGS_JSON` as the generic fallback args-prefix handle. Provider-specific variables take precedence over fallback variables.

Provider-helper collection probes are configured separately from helper execution assets. Provider-specific collection-probe executable handles are `COMATH_AGENT_ADAPTER_OSISO_OCI_COLLECTION_PROBE`, `COMATH_AGENT_ADAPTER_OSISO_NIX_COLLECTION_PROBE`, `COMATH_AGENT_ADAPTER_OSISO_FIREJAIL_COLLECTION_PROBE`, `COMATH_AGENT_ADAPTER_OSISO_WINDOWS_APPCONTAINER_COLLECTION_PROBE`, and `COMATH_AGENT_ADAPTER_OSISO_MACOS_SANDBOX_EXEC_COLLECTION_PROBE`, with `COMATH_AGENT_ADAPTER_OSISO_PROVIDER_HELPER_COLLECTION_PROBE` as a host-only fallback. Optional fixed collection-probe args prefixes may be configured through `COMATH_AGENT_ADAPTER_OSISO_OCI_COLLECTION_PROBE_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_NIX_COLLECTION_PROBE_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_FIREJAIL_COLLECTION_PROBE_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_WINDOWS_APPCONTAINER_COLLECTION_PROBE_ARGS_JSON`, or `COMATH_AGENT_ADAPTER_OSISO_MACOS_SANDBOX_EXEC_COLLECTION_PROBE_ARGS_JSON`, with `COMATH_AGENT_ADAPTER_OSISO_PROVIDER_HELPER_COLLECTION_PROBE_ARGS_JSON` as fallback. A configured collection probe is executed by `comathd` with `shell=false`, fixed argv/env, and disabled-network proof-authority metadata; stdout JSON must bind the current project, collection, helper execution, runner, launch, adapter, backend, provider, helper exit code, helper stdout/stderr/transcript hashes, host-validation id/path/hash, provider host-capability probe id/path/hash, observed host-capability status, and `provider_host_capability_bound=true` before it can produce canonical OS-isolation evidence. Task203 also requires complete collection facts to carry `provider_tool_execution_witness` bound to the service-owned probe executable hash, service-derived profile hash, fixed argv template hash, current helper transcript hash, provider, collection, helper execution, runner, launch, disabled network policy, and `proof_authority="none"`. Task204 requires that witness chain to carry provider-specific tool binding to a host-capability provider tool name/hash observed by the current service-owned capability probe, and readiness rechecks that current host-capability artifact binding. Task205 requires complete collection facts to carry `provider_family_os_enforcement_witness` material bound to the current provider family, host-capability artifact/tool binding, helper transcript, complete OS-enforcement facts, disabled network policy, and `proof_authority="none"`. Task206 requires that provider-family witness to carry provider-family execution profile binding through the service-derived execution kind, profile hash, and argv hash. Task207 requires complete collection facts to carry a provider-specific live probe attempt via `provider_specific_live_probe_attempt`, bound to the current provider-family profile, provider tool hashes, helper transcript, complete OS facts, disabled network policy, and non-authority boundary. The host-capability/host-validation, provider-tool witness, provider-specific tool binding, provider-family OS-enforcement witness, provider-family execution profile, and provider-specific live probe attempt are audit and provenance prerequisites only; they do not certify/complete release readiness or alter Lean authority. Invalid, incomplete, stale-host-capability, missing/mismatched provider-tool witness, missing provider-specific tool binding, missing provider-family OS-enforcement witness, missing provider-family execution profile, or missing provider-specific live probe attempt probes become replayable blockers.

Task208 adds provider-specific live probe execution binding. Provider-specific live-probe executable handles are `COMATH_AGENT_ADAPTER_OSISO_OCI_LIVE_PROBE`, `COMATH_AGENT_ADAPTER_OSISO_NIX_LIVE_PROBE`, `COMATH_AGENT_ADAPTER_OSISO_FIREJAIL_LIVE_PROBE`, `COMATH_AGENT_ADAPTER_OSISO_WINDOWS_APPCONTAINER_LIVE_PROBE`, and `COMATH_AGENT_ADAPTER_OSISO_MACOS_SANDBOX_EXEC_LIVE_PROBE`, with `COMATH_AGENT_ADAPTER_OSISO_PROVIDER_SPECIFIC_LIVE_PROBE` as a host-only fallback. Optional fixed live-probe args prefixes may be configured through `COMATH_AGENT_ADAPTER_OSISO_OCI_LIVE_PROBE_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_NIX_LIVE_PROBE_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_FIREJAIL_LIVE_PROBE_ARGS_JSON`, `COMATH_AGENT_ADAPTER_OSISO_WINDOWS_APPCONTAINER_LIVE_PROBE_ARGS_JSON`, or `COMATH_AGENT_ADAPTER_OSISO_MACOS_SANDBOX_EXEC_LIVE_PROBE_ARGS_JSON`, with `COMATH_AGENT_ADAPTER_OSISO_PROVIDER_SPECIFIC_LIVE_PROBE_ARGS_JSON` as fallback. Complete configured collection facts must carry `provider_specific_live_probe_execution` from that separately executed service-owned subprocess, binding the current provider-family profile, provider tool hashes, helper transcript, live probe tool/argv/stdout/stderr/transcript hashes, disabled network policy, and non-authority boundary. Missing or mismatched provider-specific live probe execution becomes a replayable blocker.

Default host validation also applies a service-observed provider/platform contract: Windows AppContainer requires `win32`, Firejail requires `linux`, macOS `sandbox-exec` requires `darwin`, Nix is limited to Linux/macOS, and OCI helpers remain a host-configured contract for Linux/macOS/Windows runtimes. On compatible platforms, the default validator runs the configured helper through a fixed CoMath provider-helper self-test argv/env before host validation can unlock helper execution. These helper paths and prefix arguments are host configuration, not proof evidence, are recorded publicly only by executable hash, args-prefix hash/count, self-test hashes/status, and provider/platform contract result, and are never Pi payload fields.
